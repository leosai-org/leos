# Contributing to LEOS

Thank you for helping improve LEOS.

## Before contributing

- Search existing issues and pull requests.
- Open an issue before substantial architectural, contract, schema, or
  compatibility changes.
- Keep changes focused and explain the user or operator impact.
- Never include credentials, private infrastructure details, proprietary code,
  confidential data, model weights without redistribution rights, or generated
  artifacts that cannot be reproduced.

## Developer Certificate of Origin

LEOS uses the Developer Certificate of Origin. Every commit must include a
`Signed-off-by` line confirming that you have the right to contribute the work:

```bash
git commit -s
```

By contributing, you agree that your contribution is licensed under the same
license that governs the destination repository or file unless a clearly
documented exception applies.

## AI-assisted contributions

AI-assisted work is welcome when the contributor:

- reviews and understands the submitted change;
- verifies correctness and security;
- confirms the output does not copy incompatible or proprietary material;
- discloses material AI assistance in the pull request;
- remains responsible for authorship, provenance, testing, and licensing.

Do not submit generated code or documentation whose origin or license cannot be
reasonably established.

## Quality requirements

A pull request should include:

- a clear problem statement and solution;
- tests or a reason tests are not applicable;
- documentation for changed behavior or contracts;
- migration notes for compatibility changes;
- security and privacy considerations;
- dependency and license changes, when applicable.

Run the relevant tests and validation tools before requesting review.

## Review and acceptance

Maintainers may request changes, split large proposals, reject incompatible
designs, or require additional provenance and security evidence. Acceptance is
based on project direction, quality, maintainability, compatibility, and
available reviewer capacity.

## Conduct

Participation is governed by `CODE_OF_CONDUCT.md`.
