# Support

LEOS is currently a Developer Preview maintained on a best-effort basis.

## Where to ask

- Use GitHub Discussions for architecture, usage, and design questions when
  discussions are enabled.
- Use an issue for reproducible bugs, documentation defects, and scoped feature
  requests.
- Use the project resources published at `https://leosai.org` for general
  project information.

## Before opening an issue

Include the LEOS version, operating system, deployment method, relevant service,
expected behavior, actual behavior, reproduction steps, and sanitized logs.

Do not include secrets, access tokens, private addresses, proprietary data, or
personal information.

## Security

Do not use support channels for vulnerabilities. Follow `SECURITY.md`.

## Service level

The open-source project does not currently provide guaranteed response times,
uptime, compatibility, or production support. Commercial support, hosting, or
enterprise offerings—when available—will be documented separately and do not
change the Apache-2.0 license of the public source.
