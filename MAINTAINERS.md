# Maintainers

## Project steward

- **Bad Tech Labs LLC** — release stewardship, repository administration,
  governance, security coordination, and project identity.

## Maintainer responsibilities

Maintainers are expected to:

- review changes for correctness, security, provenance, and compatibility;
- protect frozen release contracts and publication boundaries;
- require DCO sign-off and adequate testing;
- document significant decisions;
- manage coordinated vulnerability reports privately;
- avoid conflicts between public community interests and private commercial
  repositories.

Individual maintainers and repository-specific ownership are recorded through
Git history, repository permissions, and `.github/CODEOWNERS`.

The project may add named maintainers after the public organization and account
identities are finalized.
